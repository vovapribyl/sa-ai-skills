# AI Skill: Algorithm Discovery

## Назначение

`Algorithm Discovery` помогает системному аналитику выделить алгоритм решения задачи из бизнес-описания, требований, интервью, пользовательских историй или иной документации.

Навык предназначен для анализа предметной области и подготовки алгоритма до этапа проектирования.

Он **не занимается**:

- проектированием архитектуры;
- выбором технологий;
- написанием программного кода;
- проектированием API;
- моделированием данных.

Результат может использоваться как основа для дальнейшего проектирования:

- Activity Diagram;
- Sequence Diagram;
- Functional Specification;
- Use Case;
- API Design;
- SQL;
- Data Model.

---

# Когда использовать

Используйте навык, если необходимо:

- выделить бизнес-алгоритм из текстового описания;
- понять последовательность действий;
- определить точки принятия решений;
- обнаружить недостающие требования;
- подготовить материал для построения Activity Diagram.

---

# Входные данные

Навык принимает описание задачи в произвольном виде.

Например:

- бизнес-требования;
- Problem Statement;
- User Story;
- интервью;
- переписку;
- описание существующего процесса;
- функциональную спецификацию;
- файлы, диаграммы и таблицы.

---

# Результат

Навык формирует:

- цель алгоритма;
- входные данные;
- результат выполнения;
- ограничения;
- последовательность действий;
- точки принятия решений;
- исключительные ситуации;
- псевдокод;
- возможные оптимизации;
- вопросы для уточнения;
- оценку полноты алгоритма;
- рекомендацию следующего этапа анализа.

---

# Принципы работы

Навык придерживается следующих правил.

- Использует только информацию из входных данных.
- Не придумывает отсутствующие требования.
- Не делает скрытых предположений.
- Не смешивает бизнес-логику с технической реализацией.
- Все неопределенности выносит в вопросы для уточнения.
- Не предлагает изменения бизнес-логики.

---

# Следующий этап

После получения алгоритма рекомендуется перейти к одному из следующих навыков:

- Activity Diagram
- Sequence Diagram
- API Design
- SQL Design
- Data Model

Выбор зависит от степени детализации требований.
