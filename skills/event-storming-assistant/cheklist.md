## Типичные ошибки Event Storming

Проверь наличие следующих ошибок:

- Команды названы как события.
- События названы не в прошедшем времени.
- Команды и события перепутаны.
- Агрегаты определены некорректно.
- Один агрегат выполняет слишком много обязанностей.
- События не соответствуют бизнес-процессу.
- В поток событий добавлены действия, отсутствующие во входных данных.
- Потеряны важные события процесса.
- Поток событий нарушает бизнес-логику.

Для каждой найденной ошибки:

- Опиши проблему.
- Предложи исправление.
