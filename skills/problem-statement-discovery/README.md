# Problem Statement Discovery

Подготавливает структурированную постановку задачи на основе описания проблемы.

Скилл предназначен для ситуаций, когда заказчик описывает проблему в свободной форме: на встрече, в переписке, в обращении или документе, а аналитику необходимо преобразовать это описание в понятный и структурированный документ для дальнейшего анализа.

## Возможности

- выделяет фактическую проблему;
- описывает контекст возникновения;
- фиксирует симптомы;
- определяет последствия;
- выделяет ограничения;
- формирует ожидаемый результат;
- выявляет противоречия;
- показывает недостающую информацию;
- оценивает полноту исходных данных;
- формирует вопросы для уточнения;
- рекомендует следующий этап аналитической работы.

## Особенности

Скилл сознательно ограничивает собственные выводы.

Он:

- не придумывает требования;
- не предлагает решения;
- не проектирует архитектуру;
- не описывает реализацию;
- не делает предположений без явного указания.

Если информации недостаточно, скилл фиксирует пробелы и формирует вопросы для дальнейшего интервью.

## Когда использовать

Используйте этот скилл, если необходимо:

- разобрать обращение пользователя;
- проанализировать описание проблемы;
- обработать расшифровку встречи;
- структурировать переписку с заказчиком;
- подготовиться к интервью с бизнесом;
- сформировать стартовую постановку задачи перед сбором требований.

## Типичный процесс

```text
Описание проблемы
        │
        ▼
Problem Statement Discovery
        │
        ▼
Структурированная постановка задачи
        │
        ▼
Следующий этап анализа
```

## Следующие навыки

После использования данного скилла обычно переходят к одному из следующих этапов:

- Business Requirements Discovery
- Business Process Discovery
- Domain Model Discovery
- Algorithm Discovery
- Data Analysis

Выбор зависит от того, какая информация отсутствует и какой аспект системы необходимо исследовать дальше.
